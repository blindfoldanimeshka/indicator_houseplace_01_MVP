# Миграции базы данных

Каждое изменение production-схемы добавляется отдельным SQL-файлом с UTC-меткой
в имени, например `202607190001_secure_chat_creation.sql`. Существующий
`../schema.sql` — только историческая начальная схема; он не является способом
накатывать изменения на работающий проект.

Перед применением миграции обязательны:

1. проверка на локальном или preview-проекте Supabase;
2. тесты RLS минимум от лица двух пользователей;
3. проверка rollback-плана без удаления боевых данных.

Первая миграция должна закрыть прямые вставки в `chats` и
`chat_participants`, затем добавить атомарную RPC-функцию создания чата, как
описано в `docs/technical-specification-roadmap.md`.
