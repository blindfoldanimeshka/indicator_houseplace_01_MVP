# Backend — «напрямую» MVP

## Текущее состояние

**Бэкенд: Supabase.** Auth, PostgreSQL, Realtime — из коробки.

- Клиент: `src/lib/supabase.js`
- SQL-схема: `supabase/schema.sql`
- Auth: email + пароль через `supabase.auth`
- Данные: таблицы `users`, `listings`, `chats`, `chat_participants`, `messages`

### Было: localStorage-shim (`src/storageShim.js`)

Заглушка для локальной разработки без бэкенда. Больше не используется.

## Что нужно для реального бэкенда

### 1. Аутентификация

| Компонент | Назначение |
|---|---|
| Регистрация / логин | Привязка аккаунта к пользователю |
| JWT / session tokens | Авторизация HTTP-запросов |
| Верификация телефона / email | Защита от фейковых аккаунтов |

### 2. REST API

| Endpoint | Метод | Описание |
|---|---|---|
| `POST /auth/register` | POST | Регистрация |
| `POST /auth/login` | POST | Логин |
| `GET /listings` | GET | Лента (параметры: type, city, rooms, maxPrice) |
| `POST /listings` | POST | Создание объявления |
| `DELETE /listings/:id` | DELETE | Удаление (только свой) |
| `GET /chats` | GET | Список чатов текущего пользователя |
| `GET /chats/:chatId` | GET | Сообщения чата |
| `POST /chats/:chatId/messages` | POST | Отправка сообщения |

### 3. Реалтайм

Текущий поллинг (4 сек) заменить на:

- **WebSockets** (Socket.IO / ws) — для чата в реальном времени
- **SSE** — альтернатива для push-уведомлений

### 4. Модерация

- Автоматическая проверка на спам
- Ручная модерация по жалобам
- Ограничение числа объявлений на пользователя

### 5. Монетизация (планы)

- Платное размещение / продвижение объявлений
- Комиссия с успешных сделок
- Premium-функции (выделение в ленте, приоритет в поиске)

## Варианты стека

| Вариант | Плюсы | Минусы |
|---|---|---|
| **Supabase** | Auth + DB + Realtime из коробки | Привязка к экосистеме |
| **Firebase** | Быстрый старт, Realtime DB | Vendor lock-in, цены растут |
| **Node.js + Express + PostgreSQL** | Полный контроль | Нужно писать всё самому |
| **Go + Fiber + PostgreSQL** | Производительность | Высокий порог входа |

## Текущие ограничения

| Проблема | Последствие |
|---|---|
| Нет уникальных ID пользователей | Профиль теряется при очистке браузера |
| Нет шаринга между браузерами | Объявления видны только в одном браузере |
| Нет прав доступа | Любой может удалить любое объявление |
| Нет серверной валидации | Формат данных зависит только от фронта |
| Чат — поллинг, не реалтайм | Задержка до 4 секунд |
