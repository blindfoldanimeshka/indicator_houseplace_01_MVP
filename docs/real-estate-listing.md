# Размещение объявления о жилье (real-estate listing)

> Статус: ✅ Реализовано (фаза 8, раздел «качественное размещение»).
> Охватывает UX формы, схему БД, интеграцию карты (Yandex Maps 3.0) и
> геокодинг адреса. Документ — руководство и спецификация одновременно.

---

## 1. Принципы UX размещения

Объявление о сдаче/аренде жилья — центральная сущность продукта. Цель:
минимум трения при заполнении, максимум пользы для тех, кто ищет.

**Правила, заложенные в форму (`ListingForm.tsx`):**

1. **Обязательное — только суть.** Тип, город, комнаты, цена. Остальное
   (площадь, описание, адрес, фото) — необязательно, но рекомендуется.
2. **Адрес не блокирует публикацию.** Поле «Адрес / ориентир» опционально:
   кто не хочет светить точный адрес — пишет район/ориентир, карта скрывается.
3. **Точная точка — по желанию.** Кнопка «Найти на карте» геокодирует адрес
   в координаты; на карте можно уточнить точку кликом (drag-маркер).
4. **Фото — отдельный шаг после создания.** Сначала создаётся объявление,
   потом прикрепляются фото (≤10, валидация типа/размера в `PhotoUploader`).
5. **Мгновенная обратная связь.** Ошибки валидации под полями; геокодинг
   показывает «Точка: lat, lng»; при провале — понятный русский текст.

## 2. Модель данных

Миграция `202607190007_listing_address.sql` (накатана на `MVP-House`):

```sql
ALTER TABLE public.listings
  ADD COLUMN address    text NOT NULL DEFAULT '' CHECK (char_length(address) <= 300),
  ADD COLUMN lat        double precision,
  ADD COLUMN lng        double precision;

ALTER TABLE public.listings
  ADD CONSTRAINT listings_coords_check
  CHECK (
    (lat IS NULL AND lng IS NULL)
    OR (lat BETWEEN -90 AND 90 AND lng BETWEEN -180 AND 180)
  );
```

| Поле | Тип | Назначение |
|---|---|---|
| `address` | text | Свободный адрес/ориентир (до 300 симв). |
| `lat` / `lng` | double precision | Координаты метки. `NULL` = точный адрес не указан. |
| `coords_check` | constraint | Координаты валидны ИЛИ обе NULL. |

Существующие поля (`city`, `rooms`, `price`, `area`, `description`, `status`,
`deleted_at`) не тронуты — обратная совместимость сохранена.

## 3. Карта (Yandex Maps 3.0 / ymaps3)

### Почему Yandex, а не 2GIS
- Нативный **геокодер** (адрес → координаты) в том же экосистеме, без
  отдельного сервиса.
- Официальная поддержка **React** через динамическую загрузку скрипта.
- РФ-юрисдикция (важно в контексте `03-152fz.md`).
- 2GIS — равноценная альтернатива; переключение потребует замены только
  `MapView.tsx` + `geocode.ts` (интерфейс `GeocoderResult` тот же).

### Архитектура интеграции

```
src/
├── lib/env.ts              — VITE_YANDEX_MAPS_KEY (опционален)
├── features/listings/
│   ├── geocode.ts          — geocodeAddress(): адрес → {lat, lng, address}
│   ├── MapView.tsx         — lazy-load ymaps3, маркер, режим selectable
│   ├── ListingForm.tsx     — поле адреса + «Найти на карте» + карта
│   └── ListingDetail.tsx   — карта с меткой в конце карточки
└── types/ymaps.d.ts        — типы window.ymaps3
```

**`MapView`** — единый компонент для всех случаев:
- `lat/lng` обязательны для показа; иначе «Точный адрес не указан».
- `selectable` + `onSelect` — режим выбора точки кликом (в форме).
- Ленивая загрузка SDK по `VITE_YANDEX_MAPS_KEY`; повторный вызов
  переиспользует промис (`loadPromise`).
- Ключ отсутствует → «Карта отключена» (не ломает объявление).

### Настройка (требуется ключ от пользователя)
1. Получить **JavaScript API и HTTP Геокодер** в [cabinet.developer.tech]
   (бесплатно для некоммерческих/тестовых нагрузок).
2. Добавить в `.env`:
   ```
   VITE_YANDEX_MAPS_KEY=ваш_ключ
   ```
3. Перезапустить dev-сервер.

> Без ключа объявления работают штатно; карта просто не рендерится.

## 4. Геокодинг (`geocode.ts`)

```ts
export interface GeocoderResult { lat: number; lng: number; address: string }

export async function geocodeAddress(query: string): Promise<GeocoderResult | null>
```

- Вызывает `https://geocode-maps.yandex.ru/v1?apikey=...&geocode=...&lang=ru_RU`.
- Возвращает `null` при отсутствии ключа / ошибке сети / пустом результате.
- Формат запроса в форме: `${city}, ${address}` (город сужает поиск).

## 5. Поток создания объявления (happy path)

1. Пользователь → «+ Новое» → `ListingForm`.
2. Заполняет тип/город/комнаты/цену (обязательно).
3. *Опционально:* пишет адрес → «Найти на карте» → метка появляется на
   мини-карте → при желании кликает на карте, уточняя точку.
4. «Опубликовать» → `createListing` сохраняет все поля (вкл. `address`,
   `lat`, `lng`).
5. Форма переключается в режим загрузки фото (`PhotoUploader`).
6. «Готово» → возврат в «Мои».

На детальной странице (`ListingDetail`) в конце карточки:
- блок «Адрес» (если `address` не пуст),
- блок «На карте» с `MapView` (если `lat/lng` не NULL).
