# План: Что остаётся вне кода

Этот каталог (`docs/out-of-code/`) закрывает 4 пункта из итогового списка
проекта, которые **требуют внешних ключей, решений пользователя или
юридической проверки** и поэтому не могут быть сделаны автономно в коде.

Каждый пункт раскрыт в отдельном файле с опорой на:
- `docs/progress.md` (актуальный статус фаз и заделов),
- `docs/backend.md`, `docs/db.md`, `docs/frontend.md` (архитектура),
- актуальную документацию Supabase (Auth SMTP, Edge Functions/секреты) через Context7.

## Контекст из `docs/progress.md` (почему эти пункты здесь)

- **Фазы 0–7 завершены.** Стек окончательно **Supabase** (roadmap приоритетнее
  `docs/plan.md`, который предлагал Next.js + Telegram Login).
- **Фаза 3b уже сделала задел:** шаблоны-заглушки Privacy/Terms, чекбокс
  согласия на ПДн при регистрации, `resetPasswordForEmail`, клиентский
  `email-confirmation guard`. Это снимает часть работы в 02 и 03.
- **Остающиеся блокеры (оригинальный TODO):** CAPTCHA (ждёт провайдер+ключи),
  собственный SMTP (ждёт решение), 152-ФЗ (тексты ждут юриста).
- **env-стек проекта:** использует префикс `VITE_` (см. `.env` /
  `src/lib/env.ts` с Zod-валидацией) — новые публичные ключи заводим туда же.

## Рекомендуемый порядок (связанный, а не параллельный)

```
1. 04 (beta-инвайты) ──► пока база мала, решаем 03.1 (инфраструктура ПДн)
       │
       ├─ 02 (SMTP) ──► нужен, чтобы beta-юзеры могли подтвердить email
       │                (иначе email-confirmation guard их блокирует)
       │
       └─ 01 (CAPTCHA) ──► на бете можно выключить; включить Turnstile
                           перед публичным запуском (или Yandex для 152-ФЗ)
```

**Логика:** бета (04) не может стартовать без рабочего подтверждения email →
нужен SMTP (02). А пока пользователей мало — самое время решить вопрос
инфраструктуры ПДн (03, раздел 1), пока не больно переезжать. CAPTCHA (01)
наименее критична для беты, но обязательна к публичному запуску.

## Что НЕ требует внешних решений (можно кодить сразу)

Из смежных вещей, упомянутых в планах, код можно писать уже сейчас, без
ожидания провайдеров/юристов:

- Кнопка «Удалить аккаунт и мои данные» (права субъекта 152-ФЗ) — `04`/`03`.
- Поле «Инвайт-код» + валидация на клиенте — `04`.
- Кнопка повторной отправки письма на экране ожидания подтверждения — `02`.
- Структура черновиков Privacy/Terms под юриста — `03`.
- Таблица `invites` + миграция + RLS-функция `is_invite_valid` — `04`.
