# напрямую — indicator_houseplace_01_MVP

MVP-прототип сервиса аренды жилья напрямую от собственников, без агентств.

## Идея

- Собственники публикуют объявления «Сдаю».
- Арендаторы публикуют «обратные» объявления «Ищу» с желаемыми параметрами —
  так собственники тоже могут найти подходящего арендатора, а не только ждать звонков.
- Есть встроенный чат между сторонами по конкретному объявлению.

## Стек

- React 18 + Vite
- Tailwind CSS
- lucide-react (иконки)

## Запуск локально

```bash
npm install
npm run dev
```

Откроется на `http://localhost:5173`.

Сборка для продакшена:

```bash
npm run build
npm run preview
```

## Хранение данных

Приложение изначально написано под `window.storage` — API хранилища ключ-значение,
который предоставляет среда Claude.ai artifacts (личные + общие данные, без бэкенда).

Чтобы проект можно было запускать и тестировать локально, в `src/storageShim.js`
добавлена заглушка: если `window.storage` не предоставлен хостом, она эмулирует
тот же интерфейс поверх `localStorage`. Это **только для локальной разработки** —
`localStorage` не шарится между разными пользователями и браузерами.

**Перед реальным запуском в продакшен** это нужно заменить на настоящий бэкенд:
свою API + базу данных, либо BaaS-решение (Supabase/Firebase и т.п.).

## Известные ограничения MVP (не решены намеренно, для скорости прототипа)

- Нет верификации, что автор объявления — реальный собственник, а не агент.
- Нет полноценной аутентификации (профиль — это просто имя, введённое один раз).
- Нет модерации объявлений.
- Чат обновляется поллингом (раз в 4 сек), а не через сокеты/realtime.
- Нет юнит-экономики/монетизации — это чисто продуктовый прототип для проверки гипотезы.

## Структура
